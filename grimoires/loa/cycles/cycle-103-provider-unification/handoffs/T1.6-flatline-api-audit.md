## Migration pattern (chat sites)

**Before** (representative — `flatline-proposal-review.sh::call_gpt_review`):

```bash
local response
if declare -f call_api_with_retry &>/dev/null; then
    response=$(call_api_with_retry "${OPENAI_API_BASE:-https://api.openai.com}/v1/chat/completions" "POST" \
        "$(jq -n --arg prompt "$prompt" --arg model "$GPT_MODEL" '{
            model: $model,
            messages: [{role: "user", content: $prompt}],
            temperature: 0.2,
            max_tokens: 500
        }')" "$TIMEOUT_SECONDS")
else
    # fallback raw-curl path with auth tempfile + endpoint_validator__guarded_curl
    local _curl_cfg
    _curl_cfg=$(write_curl_auth_config "Authorization" "Bearer ${OPENAI_API_KEY:-}") || …
    printf 'header = "Content-Type: application/json"\n' >> "$_curl_cfg"
    response=$(endpoint_validator__guarded_curl \
        --allowlist "$FLATLINE_PROVIDERS_ALLOWLIST" \
        --config-auth "$_curl_cfg" \
        --url "${OPENAI_API_BASE:-https://api.openai.com}/v1/chat/completions" \
        -s --max-time "$TIMEOUT_SECONDS" \
        -X POST \
        -d "$(jq -n --arg prompt "$prompt" --arg model "$GPT_MODEL" '{
            model: $model,
            messages: [{role: "user", content: $prompt}],
            temperature: 0.2,
            max_tokens: 500
        }')" 2>/dev/null)
    rm -f "$_curl_cfg"
fi

if [[ -z "$response" ]]; then
    log_error "Empty response from GPT"
    return 4
fi

local result
result=$(echo "$response" | jq -r '.choices[0].message.content // ""' 2>/dev/null)
```

**After**:

```bash
local result
if ! result=$(call_flatline_chat "$GPT_MODEL" "$prompt" "$TIMEOUT_SECONDS" 500); then
    log_error "model-invoke failed for GPT proposal review"
    return 4
fi

if [[ -z "$result" ]]; then
    log_error "Empty response from GPT"
    return 4
fi
```

The helper is in `.claude/scripts/lib-curl-fallback.sh:472–517`. Each migrated
script sources it.

## The embeddings exception (site #6)

`flatline-semantic-similarity.sh::get_embedding` calls
`POST https://api.openai.com/v1/embeddings` and parses
`response.data[0].embedding` (a 1536-dim float vector for
`text-embedding-3-small`).

**Why deferred:**

1. **Cheval has no embeddings substrate.** `.claude/adapters/cheval.py`
   dispatches `CompletionRequest` (chat) only. The provider adapter base
   class `loa_cheval/providers/base.py` has a single `complete()` method —
   no `embed()` method. Adding embeddings is a substantial substrate
   expansion (new types, new exit codes, new normalization), not a T1.6
   sub-task.
2. **Different shape, different consumer contract.** Chat returns text;
   embeddings return a float vector. The caller-side parsing
   (`get_embedding_from_index`, cosine-similarity computation) is
   embedding-specific. A 1:1 replacement isn't possible without expanding
   `model-invoke`'s output contract.
3. **Cycle-103 invariant scope.** M2 reads "Flatline → cheval". Sprint plan
   risk register did not call out embeddings. Closing M2 for chat is the
   substantive coverage; embeddings stay direct and the call still passes
   through `endpoint_validator__guarded_curl` with the providers allowlist
   (cycle-099 sprint-1E.c.3.b).

**Tracking:** filed for cycle-104+ via this audit doc. When cheval adds an
`embed()` method, migrate site #6 then.
