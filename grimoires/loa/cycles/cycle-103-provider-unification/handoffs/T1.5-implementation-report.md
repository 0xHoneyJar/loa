## Tasks Completed

### T1.5 — Extend cheval CLI with `--mock-fixture-dir`

**File modified:** `.claude/adapters/cheval.py`

**Changes (4):**

1. **`_sanitize_fixture_model_id()`** (`cheval.py:280-288`) — strips path-hostile chars from model IDs (`:`, `/`, etc.) so e.g. `gpt-5.5-pro` → `gpt-5.5-pro` (no-op) and `openai:gpt-5.5` → `openai_gpt-5.5`. Conservative allowlist (alnum + `_-.`).

2. **`_load_mock_fixture_response(fixture_dir, provider, model_id)`** (`cheval.py:291-415`) — returns a `CompletionResult`. Filename precedence: `<provider>__<sanitized>.json` then `response.json`. Validates JSON shape, required fields, type-pins each field. Path-traversal defense via realpath containment (`cheval.py:329-331`).

3. **`cmd_invoke` dispatch branch** (`cheval.py:642-660`) — before `invoke_with_retry` / `adapter.complete`, check `getattr(args, "mock_fixture_dir", None)`. If set: skip provider dispatch, call `_load_mock_fixture_response`, continue through normal MODELINV emit + stdout output.

4. **argparse flag** (`cheval.py:820-832`) — `--mock-fixture-dir <path>` registered on main parser with explanatory help text referencing AC-1.2 and IMP-006.

**Design choices:**

- **Filename precedence: per-(provider, model) wins over `response.json`.** Lets BB tests drop a single `response.json` for cross-provider parity tests AND override per-provider when the multi-model paths diverge (cross-model consensus scenarios).
- **Fixture mode preserves budget hook + input gate + MODELINV emit.** The fixture replaces ONLY the adapter call — every other code path runs identically to a real invocation. Tests verifying budget enforcement / input-gate refusal / audit-emit behavior continue to work in fixture mode.
- **IMP-006 normalization on load, not on save.** `latency_ms` defaults to 0, `interaction_id` to None, `usage.source` to "actual" unless the fixture explicitly pins them. This lets operators re-record fixtures (e.g., for a new model) without re-normalizing — defaults handle the common case automatically.
- **Path traversal defense via realpath containment.** Each candidate path is realpath-resolved, then verified to be inside the realpath-resolved fixture_dir. Catches the symlink-escape attack (`response.json` → `../../etc/passwd`). Test `test_symlink_escaping_dir_is_not_followed` pins this behavior.
- **Validate types explicitly.** Each non-string-`content` / non-int-token / non-list-`tool_calls` / non-string-`thinking` / non-string-`interaction_id` raises `InvalidInputError` with a precise message. Cheap defensive programming for hand-edited fixtures.

- **Karpathy-aligned scope.** T1.5 ships only the cheval-side flag + fixture loader + tests. No refactor of `cmd_invoke`'s existing structure (the fixture branch is one if/else around the existing dispatch). No changes to adapter modules. No changes to retry/budget logic.

The pytest does this at `test_mock_fixture_dir.py:303`.

## Known Limitations

1. **Existing BB per-provider test suites NOT yet migrated.** The three `__tests__/{anthropic,openai,google}.test.ts` files still test the legacy adapters directly. AC-1.2's "All three TS adapter test suites... pass against the new delegate using mocked-fixture comparisons" closes when **T1.4** migrates `adapter-factory.ts` to return the delegate AND those test suites either (a) retire, or (b) get rewritten as delegate-via-fixture tests. T1.5 builds the substrate; T1.4 is the migration step.
2. **Sanitization is path-conservative, not Unicode-safe.** `_sanitize_fixture_model_id` keeps only ASCII alphanumeric + `_-.`. A model ID with a Unicode letter (none ship today; all current model IDs are ASCII) would have each non-ASCII char collapse to `_`. Acceptable for current operator workflows; revisit if Unicode model IDs land.
3. **No fixture-mode-aware MODELINV emit signal.** The MODELINV envelope emitted from a fixture call looks identical to a real-call envelope, with `models_succeeded` containing the resolved binding's `provider:model`. Operators auditing the chain cannot distinguish fixture calls from live calls without consulting `result.metadata.mock_fixture`. **Flagged for Sprint 3 AC-3.2 consideration** — the metadata is available but the envelope emitter doesn't currently surface it. Not blocking AC-1.2.
4. **`response.json` lookup is per-directory only — no nested lookup.** If a test scaffold puts fixtures in subdirectories (e.g., `fixtures/anthropic/response.json`), the caller MUST pass the full subdir path as `--mock-fixture-dir`. The flag is a single directory, not a fixture-set root. Documented in argparse help text.
5. **SDD §5.1 CLI contract still shows `--timeout-seconds` + `--json-out`.** T1.2's report flagged this divergence. T1.5 did NOT update the SDD; flagged for Sprint 1 SDD-correction sweep alongside T1.2's note.
