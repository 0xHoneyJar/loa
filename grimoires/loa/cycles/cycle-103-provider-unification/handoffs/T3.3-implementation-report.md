## Tasks Completed

### T3.3 — sanitize helper + wiring

**New module:** `.claude/adapters/loa_cheval/redaction/sanitize.py` (~95 lines).

The helper is a layered pipeline:

```python
def sanitize_provider_error_message(message: Optional[str]) -> str:
    if message is None:
        return ""
    if not isinstance(message, str):
        message = str(message)

    text = _normalize_for_sanitize(message)  # NFKC + zero-width strip
    text = _log_redact(text)                 # AKIA/PEM/Bearer/URL/query
    text = _SK_ANT_RE.sub("[REDACTED-API-KEY-ANTHROPIC]", text)
    text = _SK_OPENAI_RE.sub("[REDACTED-API-KEY-OPENAI]", text)
    text = _AIZA_RE.sub("[REDACTED-API-KEY-GOOGLE]", text)
    return text
```

Patterns:

- `_SK_ANT_RE = re.compile(r"sk-ant-[A-Za-z0-9_\-]{24,}")` — Anthropic
- `_SK_OPENAI_RE = re.compile(r"sk-(?!ant-)[A-Za-z0-9_\-]{24,}")` — OpenAI w/ Anthropic-disambiguation negative lookahead
- `_AIZA_RE = re.compile(r"AIza[A-Za-z0-9_\-]{35}(?![A-Za-z0-9_\-])")` — exactly 39 chars total, no overshoot
- `_ZERO_WIDTH = re.compile("[<ZW chars>]")` — mirrors T3.7 + cycle-099 sprint-1E.c.3.c

The 24-char minimum on `sk-ant-*` / `sk-*` body is generous — real API
keys are 48-96 chars, but the conservative floor catches abbreviated
shapes without false-positive on `sk-ant-` mentions in documentation
strings.

Anthropic / OpenAI disambiguation: the `(?!ant-)` negative lookahead on
the OpenAI pattern prevents `sk-ant-XXX` from double-matching (would
otherwise be redacted by both patterns producing
`[REDACTED-API-KEY-OPENAI]` over the already-redacted output).

Order: log-redactor runs BEFORE the new patterns so URL-embedded
provider keys get the URL-userinfo / query-string handling first.

**Wiring at exception-construction sites:**

1. **`_extract_error_message`** — three adapters each have one of these.
   Pattern: take the upstream `error.message` field, return through
   sanitize. Every retry-layer call to `_raise_for_status` then routes
   through `_extract_error_message`, so EVERY exception constructed from
   an HTTP error response body gets sanitized — `RateLimitError`,
   `ProviderUnavailableError`, `InvalidInputError` all inherit the
   redaction transparently.

2. **Streaming-error wrap sites** — three adapters each have an
   `except ValueError as parse_err: raise InvalidInputError(...)` block.
   Sanitize wraps the `f"<provider> streaming error: {parse_err}"`
   format string. `parse_err.args[0]` comes from mid-stream provider
   error events (Anthropic `error` events, OpenAI `response.failed`,
   Google SAFETY/RECITATION blocks) — all contain upstream-derived
   strings.

3. **`retry.py` `RetriesExhaustedError`** — two construction sites:
   global-budget exit (line 163) and retry-loop exit (line 287). Both
   carry `last_error` strings that accumulate from `str(e)` of caught
   exceptions across attempts. Sanitize wraps `last_error` before
   passing.

## Technical Highlights

- **Karpathy-aligned: minimum-scope wiring.** No new abstractions beyond the helper module. Wires at the existing extraction sites (already centralized in each adapter) plus the two retry.py exit points. Total surface: 1 new file + 4 modified files; ≤6 lines per wire site.
- **Layered on top of `log-redactor.py`, not a replacement.** AKIA / PEM / Bearer / URL-userinfo / query-string patterns are already pinned by the cycle-099 sprint-1E.a + cycle-102 sprint-1D parity tests (Python + bash twin). T3.3 reuses that proven path via `importlib`-based load and adds three provider-specific patterns on top. If the underlying redactor evolves, this helper inherits the changes for free.
- **NFKC + zero-width strip is the load-bearing defense.** Without normalization, an attacker emitting `Ａ Ｉ ｚ ａ ＿ ＿...` (fullwidth) or `sk-an​t-XXX` (zero-width insertion) would bypass the literal-prefix regex. The same defense pattern shipped in cycle-099 sprint-1E.c.3.c (Unicode-glob bypass closure in `check-no-raw-curl.sh`) and cycle-103 sprint-3 T3.7 (`_GATE_BEARER` normalization). T3.3 reuses that exact pattern.
- **Anthropic-vs-OpenAI disambiguation via negative lookahead.** Without `(?!ant-)`, the OpenAI pattern would match `sk-ant-...` as well, producing `[REDACTED-API-KEY-OPENAI]` over the Anthropic redaction's output. The lookahead is the minimum-state way to disambiguate.
- **`AIza` pattern has explicit length anchor.** `AIza[A-Za-z0-9_\-]{35}(?![A-Za-z0-9_\-])` matches EXACTLY 39 chars (4 + 35) — Google's documented key length. The negative lookahead prevents matching 40-char strings as "valid Google keys with an extra char" — preserving the documented shape exactly. `TestNegativeControls::test_aiza_wrong_length_not_redacted` pins this.
- **Wired at retry's `RetriesExhaustedError` final-cause chain.** Sprint.md L218 calls out `retry.py` specifically. The two construction sites are the only paths that produce `RetriesExhaustedError`; both now sanitize `last_error` before passing. This means cheval.py's stderr emission of the final-cause string is guaranteed sanitized.
- **`from loa_cheval.redaction import ...` inside function body.** Adapter `_extract_error_message` uses a lazy import to avoid circular-import risk with `loa_cheval.redaction → log-redactor.py loading at module load`. The cost (single attribute lookup per call) is negligible compared to the JSON-parse work the adapter does anyway.

## Known Limitations

1. **`bedrock_adapter.py` not wired.** The sprint is scoped to anthropic / openai / google; bedrock is a separate adapter with its own `_extract_error_message`. AWS keys are already covered by the AKIA pattern from log-redactor (transparently inherited), but if Bedrock ever surfaces provider-keys-as-AKIA-impostors, this gap matters. Add the wire when Bedrock comes into cycle scope.
2. **No metric on sanitization-trip count.** A trip raises `[REDACTED-*]` into the output but doesn't emit a metric / log line. If operators want to see "how often was a provider API key actually scrubbed", add a counter. Not blocking — T3.3 closes the leak path, not the observability gap.
3. **Pattern set is closed.** Cohere / Mistral / xAI / Together API keys are not pinned. If a future cycle adds an adapter for those providers, extend the pattern set. The architecture supports it cleanly: add one `_<PROVIDER>_RE` + one `re.sub(..., sentinel, text)` line.
4. **Two-level URL encoding not handled.** Mirrors T3.7 known limitation — `%2520Bearer%2520token` would not double-decode. Single-pass redaction is the intentional amplification defense.
