## Tasks Completed

### T1.6 — Migrate Flatline direct-API calls to model-invoke

**Approach:**

1. **Enumerate.** Grep for provider URL substrings across `flatline-*.sh`. 6 sites total: 5 chat/completion + 1 embeddings. (Full audit at `handoffs/T1.6-flatline-api-audit.md`.)
2. **Add generic helper.** `call_flatline_chat(model, prompt, timeout, max_tokens)` in `lib-curl-fallback.sh`. Returns content text directly (no provider-specific `.choices[0].message.content` extraction; the caller does its own JSON parsing). Writes prompt to a tempfile passed via `--input` so prompt content stays out of argv (AC-1.8). Uses `--agent flatline-reviewer` as a binding holder; `--model` override pins the actual provider+model.
3. **Migrate each call site.** Replace the if/else (retry-helper-or-curl-fallback) block + provider response-shape extraction with a single helper call. The downstream JSON-extraction-and-schema-validation logic is unchanged.
4. **Source the helper.** Each flatline script gets `source "$SCRIPT_DIR/lib-curl-fallback.sh"` after its existing security/api-resilience sourcing.
5. **Bats coverage.** 11 tests covering argv validation, happy path, argv shape, secret-not-in-argv (AC-1.8), error propagation, missing-binary handling, temp-file cleanup (success AND error), and end-to-end via real `python3 cheval.py --mock-fixture-dir`.

**Behavioral change documented in audit doc:** the per-call temperature override (was 0.2 or 0.3 depending on site) is replaced by the agent binding's default (0.3 for `flatline-reviewer`). Within the deterministic-review band; flagged as accepted-deferred should a future investigation surface drift.

### Why I did NOT migrate the embeddings call

Site #6 (`flatline-semantic-similarity.sh::get_embedding`) calls
`POST https://api.openai.com/v1/embeddings` and parses
`response.data[0].embedding` (a 1536-dim float vector). Cheval's
`CompletionRequest` / `CompletionResult` types model chat completion only —
no `embed()` method exists on the adapter base class. Adding an embeddings
substrate is a substantial expansion (new types, new exit codes, new
normalization, new caller contract) that doesn't belong in T1.6.

The embeddings call still flows through `endpoint_validator__guarded_curl`
with the providers allowlist (cycle-099 sprint-1E.c.3.b), so it remains
security-equivalent — just not yet substrate-unified. Tracked in the audit
doc as a cycle-104+ task.

## Known Limitations

1. **Embeddings call NOT migrated.** Documented in audit doc + AC table above. Cheval needs an embeddings substrate before this site can route through `model-invoke`. Out of cycle-103 scope.
2. **Temperature override lost.** Previously each call site had per-call temperature (0.2 for review tasks, 0.3 for extraction). Now all sites use the `flatline-reviewer` binding's default temperature (0.3). Within deterministic-review band; flagged as accepted-deferred.
3. **`call_api_with_retry` retry policy lost on migrated paths.** The legacy code branched into `call_api_with_retry` (with backoff + circuit breaker) OR raw curl. After migration, retries live entirely on cheval's side (`loa_cheval/providers/retry.py`). Cheval's retry is provider-aware and integrates with the model-invoke MODELINV envelope. **Net behavior should be EQUAL OR BETTER** since cheval's retry knows about provider-specific failure classes. Not regression-tested live in this task.
4. **`flatline-reviewer` agent used for all migrated sites.** Per-site agent selection (e.g., `gpt-reviewer` for OpenAI, `jam-reviewer-claude` for Anthropic) would let cheval's per-agent persona apply. For T1.6's minimum scope, all sites use one agent; the `--model` override pins the provider. If future investigation surfaces persona drift, the helper's signature can be extended with an `agent` arg.
5. **Bats e2e test depends on `python3` on PATH.** Skips gracefully if absent. Confirmed working under this repo's `python3` 3.13.5.
6. **Latent: `claude-opus-4-7` hyphen-form not in alias map.** See Technical Highlights. Not introduced by T1.6 but discovered during T1.6 testing.
