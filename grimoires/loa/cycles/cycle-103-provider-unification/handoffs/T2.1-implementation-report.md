## Tasks Completed

### T2.1 — Empirical replay infrastructure

**File 1: `sprint-2-corpus/seed.md`** (~5KB, deterministic seed text)

The seed is technical content describing the cycle-103 architecture — content-rich enough to exercise the model's full attention, policy-neutral, deterministic across re-runs. Mentions the documented keywords (`cycle-103`, `M1/M2/M3`, `cheval`, `delegate`, `unification`) so the classifier's `full_content` keyword check has signal.

**File 2: `sprint-2-corpus/build_prompts.py`** (~110 lines)

Deterministic prompt generator. `build_prompt(target_tokens, salt)` returns a prompt sized within ±6% of target (measured: -6% / -1% / -2% / -3% / -1% at 30K/40K/50K/60K/80K respectively). The same `(target_tokens, salt)` pair always returns the same string. `build_corpus()` iterates the full 5-size × 5-salt = 25-prompt matrix. `__main__` block runs a smoke-check that prints per-size delta to stderr — used by the bats wrapper.

**File 3: `tests/replay/classifier.py`** (~140 lines)

Pure-function classifier. Three layers:

1. **`classify_trial(content) → TrialOutcome`** — single-trial classification (EMPTY / PARTIAL / FULL). `EMPTY` for None / whitespace-only. `PARTIAL` for <50 visible tokens OR ≥50 tokens but no keyword engagement. `FULL` for ≥50 tokens AND at least one keyword match (case-insensitive).
2. **`classify_replay_run(size, outcomes) → ClassificationResult`** — n-trial aggregation. Returns counters + `full_rate` + `structural_fix_viable` (boolean from `full_rate >= 0.80`).
3. **`find_safe_threshold(matrix) → int | None`** — operator-decision function. Returns the LARGEST input size at which structural-fix-viable is True, or None if no size qualifies (vendor-side path).

All three layers offline-testable. No API calls, no I/O.

**File 4: `tests/replay/test_classifier.py`** (~180 lines, 27 tests)

Offline unit tests for every classifier disposition:
- `classify_trial`: None/empty/whitespace → EMPTY; short-with-keyword → PARTIAL; long-off-topic → PARTIAL; long-on-topic → FULL; case-insensitive keyword matching; parameterized over every documented keyword.
- `classify_replay_run`: 100% full → viable; 4/5 (80%) → viable; 3/5 (60%) → not viable; mixed counts correctly counted; empty outcomes raises.
- `classify_matrix`: orders by input size ascending; each size classified independently.
- `find_safe_threshold`: returns largest viable size; None when no size qualifies; returns max when all viable.
- `ClassificationResult`: invariant — count mismatch raises.

**File 5: `tests/replay/test_opus_empty_content_thresholds.py`** (~280 lines)

Live-replay scaffold with `LOA_RUN_LIVE_TESTS=1` gate. 151 tests collected:
- 150 parametrized `test_replay_cell` (5 sizes × 5 trials × 3 thinking budgets × 2 max_tokens) — each invokes cheval with one prompt, records the trial in JSONL.
- 1 `test_disposition` — aggregates the JSONL after all cells complete and surfaces the AC-2.1 disposition (structural vs vendor-side) to stderr + a `.summary.json` file.

Output paths:
- Per-trial JSONL: `grimoires/loa/cycles/cycle-103-provider-unification/sprint-2-corpus/results-<timestamp>.jsonl`
- Disposition summary: alongside, `.summary.json`

Per-trial dataclass captures: timestamp, target/actual input tokens, salt, thinking_budget, max_tokens, cheval exit code, latency_ms, output char count, output token estimate, outcome (TrialOutcome value), 500-char stdout/stderr previews for debugging.

**File 6: `tests/replay/run_replay.sh`** (~80 lines)

Operator-facing wrapper:
- **Dry-run mode** (default): prints invocation instructions + runs `pytest --collect-only` to verify scaffold integrity. Exit 0.
- **Live mode** (`LOA_RUN_LIVE_TESTS=1`): credential pre-flight (`ANTHROPIC_API_KEY` required); 5-second budget-warning countdown ("estimated ~$3 USD"); dispatches `pytest` with `--tb=line` (readable across 150 tests) and `-p no:cacheprovider` (long run shouldn't trip cache invalidation halfway through). On failure, points operator to the partial JSONL output.

## Technical Highlights

- **Karpathy-aligned: scaffold-only.** This task ships infrastructure for an empirical experiment, not the experiment's data. The data ships when the operator runs the wrapper with API credentials and budget. Offline testability of the classifier means we can iterate the decision-rule logic without burning API calls.
- **IMP-008 corpus stability.** The same `(target_tokens, salt)` pair always returns the same prompt. An operator re-running the replay months later gets reproducible inputs — critical for comparing threshold characterizations across vendor model updates.
- **Decision-rule mechanically pinned.** AC-2.1's "≥80% full_content" rule is a single constant (`_STRUCTURAL_VIABILITY_RATE = 0.80`) at `classifier.py:33`. 27 offline tests pin the boundary behavior (exactly 80% is viable; 60% is not; mixed outcomes counted correctly).
- **Keyword check defends against hallucinated refusals.** A classifier that only counted visible tokens would mark a long generic refusal ("I cannot help with that request..." × 50 repetitions) as FULL. The keyword check (`cycle-103` / `M1` / `M2` / `M3` / `cheval` / `delegate` / `unification` — drawn from the seed) requires the model to actually engage with the prompt's content. Lowers false-positive `full_content` rate.
- **Budget-aware design.** 150 cells × ~$0.02/call ≈ $3. PRD §8.5 specifies ~$3 budget. The wrapper warns the operator before consuming any budget. The R3a risk in sprint.md ("budget overrun") is mitigated by the explicit cell count + per-cell logging.
- **`LOA_THINKING_BUDGET` env-var integration point documented.** Cheval doesn't currently expose a `--thinking-budget` CLI flag. The scaffold passes the budget via env var with a TODO comment at `test_opus_empty_content_thresholds.py:163`. When cheval grows the flag, the call site swaps env→CLI in one place.

## Known Limitations

1. **No live data.** AC-2.1 closes with an actual replay run + classifier verdict. This commit ships the scaffold; live execution is operator-deployment work. Operator instructions are in `run_replay.sh`'s dry-run output.
2. **`LOA_THINKING_BUDGET` env var is aspirational.** Cheval doesn't currently read it (no `--thinking-budget` flag exists). The replay matrix exercises three values (None, 2_000, 4_000), but until cheval respects the env var, the budget dimension of the matrix collapses to a no-op. **Future cycle:** add `--thinking-budget` to cheval CLI; the scaffold is already wired.
3. **Token-count estimation is chars/4.** Both the corpus generator and the classifier use the same `chars // 4` estimator. Anthropic's actual tokenizer is more nuanced; a real BPE tokenizer would give different counts. The estimator is consistent with cheval's own `loa_cheval.providers.base.estimate_tokens` so the input-gate semantics match across the test surface.
4. **`test_disposition` doesn't assert pass/fail.** It logs the AC-2.1 disposition to stderr + writes `.summary.json`. The operator-side decision (apply T2.2a vs T2.2b) is informed by the summary, not asserted at pytest level. Rationale: an empirical replay's outcome is data, not a binary pass — the binary is read off the summary.
5. **150-cell matrix is large.** At ~10 seconds per call (Opus on 80K input can be slower), the full replay runs ~25 minutes. Operators with limited time can run a reduced matrix (`-k "input_tokens_target == 30000"` for sanity-only) or set `TRIALS_PER_CELL=2` via a future env-var.
6. **Budget ceiling not enforced in the scaffold.** sprint.md R3a says "Halt replay after $5 budget; if no clean threshold emerges, default to vendor-side classification". The scaffold doesn't implement this halt — it relies on the operator's hand-on-the-wheel to Ctrl-C. **Future cycle:** add a cost-counter that totals `result.latency_ms` (proxy for tokens used) and exits when projected cost ≥ $5.
