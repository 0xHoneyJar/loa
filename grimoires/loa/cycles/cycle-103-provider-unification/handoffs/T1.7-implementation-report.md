## Tasks Completed

### T1.7 — Drift gate scanner + allowlist + workflow + tests

**Scanner — `tools/check-no-direct-llm-fetch.sh` (260 lines).**

Design mirrors the cycle-099 `check-no-raw-curl.sh` precedent. Differences:

| Aspect | check-no-raw-curl | check-no-direct-llm-fetch (T1.7) |
|--------|-------------------|----------------------------------|
| Languages | bash only (`.sh`/`.bash`/`.legacy` + shebang) | bash + TypeScript + Python (`.sh`/`.bash`/`.ts`/`.tsx`/`.py` + shebang) |
| Patterns | `curl`/`wget` literal-token regex | 3 provider URL substrings (literal, no boundary needed) |
| Scope | `.claude/scripts/` | `.claude/skills/` + `.claude/scripts/` |
| Exempt source | hardcoded array | external `*.allowlist` file (mode 0644) |
| Comment skip | bash only | bash `#`, TS `//`, TS/Python `*` (block-comment continuation) |

Detection order (each step skips lines that match):

1. File-type filter (`_is_in_scope`) — extension allowlist + shebang fallback. Out-of-scope extensions (`.md`, `.json`, `.yaml`, binaries, etc.) short-circuit before any read, so binary files don't pollute stderr with null-byte warnings.
2. Exempt-file filter (`_is_exempt`) — exact path-match against entries in `tools/check-no-direct-llm-fetch.allowlist`.
3. Comment-leading lines — `^[[:space:]]*(#|//|\*)`.
4. Suppression marker — `(#|//)...check-no-direct-llm-fetch: ok` (line-scoped; marker must be in a comment, not a string literal).
5. Heredoc body — provider URLs inside `<<EOF` blocks are docs, not fetches; swallowed by the heredoc-state tracker.
6. URL substring match — `api.anthropic.com|api.openai.com|generativelanguage.googleapis.com`.

The heredoc opener line is **separately scanned after the opener** for the same pattern (cycle-099 H2 precedent): `cat <<EOF >x && curl https://api.openai.com/...` correctly flags the curl.

**Allowlist — `tools/check-no-direct-llm-fetch.allowlist` (mode 0644).**

6 exempt entries with documented rationale:

1. `.claude/scripts/model-health-probe.sh` — operator-visible direct curl health probes (different semantics from app fetches).
2. `.claude/scripts/model-adapter.sh.legacy` — already legacy-marked; sunset target.
3. `.claude/scripts/flatline-semantic-similarity.sh` — embeddings (cheval has no embeddings substrate; T1.6 audit doc tracks).
4. `.claude/scripts/lib-curl-fallback.sh` — gpt-review-api `call_api()` legacy fallback path (out of cycle-103 scope; T1.6 added `call_flatline_chat()` to the same file).
5. `.claude/scripts/lib/endpoint-validator.py` — the validator's own internal code references URLs in docstrings + defensive paths.
6. `.claude/scripts/lib/model-probe-cache.py` — provider→(host, port) map for socket-level health probes (TCP handshake, not HTTP).

Each entry is paired with rationale text in the allowlist file — a PR adding an entry forces reviewer visibility on the justification.

**Workflow — `.github/workflows/no-direct-llm-fetch.yml`.**

- Triggers: `pull_request` + `push: branches: [main]`.
- Path filter: scanner + allowlist + workflow itself + every file in scan scope.
- Concurrency group cancels in-progress runs on the same ref.
- Pre-flight: verifies allowlist file is mode 0644 (sprint spec mandate). Refuses to run if mode drift is detected — defends against an operator side-loading a private looser-mode allowlist that bypasses governance.
- Step `Run drift gate`: invokes `./tools/check-no-direct-llm-fetch.sh`. Non-zero exit maps to CI failure.
- `permissions: contents: read` (least-privilege).
- Pinned `actions/checkout@<SHA>` matching the SHA used by sibling cycle-099 / cycle-102 workflows.

## Technical Highlights

- **Karpathy-aligned scope.** One scanner, one allowlist, one workflow, one bats suite. No new abstractions beyond what the sprint mandated. Followed the precedent's exact pattern so reviewers familiar with `check-no-raw-curl.sh` can audit T1.7 by diff.
- **External allowlist over hardcoded array.** Sprint spec mandates this (vs. cycle-099's hardcoded `EXEMPT_FILES` array). Trade-off: easier to edit (file-only PR) at the cost of slightly more surface for governance (the workflow mode-check (`stat -c %a == 644`) defends against private looser-mode allowlists).
- **Heredoc tracking carried over from precedent.** Bash heredocs commonly contain URL mentions in usage strings. Without the tracker, the scanner would flag docs/examples as violations. The tracker is best-effort for non-shell files (Python and TS don't have bash-style heredocs, but their absence of `<<EOF` patterns means the tracker's false-trigger rate on them is effectively zero).
- **Binary file null-byte handling.** First iteration produced bash "command substitution: ignored null byte in input" warnings when reading binaries during shebang detection. Fixed by short-circuiting on known non-script extensions (`.md`, `.json`, `.png`, etc.) BEFORE shebang probe, plus `tr -d '\0'` in the shebang probe itself. Final scan output is warning-free.
- **Real-repo scan delta-checked.** Pre-T1.7 grep returned 35+ URL matches across 14 files; post-allowlist scanner returns **zero violations**. The delta isn't a bug — it's the documented exempt set being deliberately excluded.

## Known Limitations

1. **String-interpolated URLs are not caught.** A motivated bypass like `base="https://api.${provider}.com"; fetch("${base}/v1/chat")` constructs the URL at runtime and the static scanner has no way to detect it. Mitigation: same as the cycle-099 precedent — the scanner is one layer of defense; the cheval substrate, the routing chokepoints (`ChevalDelegateAdapter`, `call_flatline_chat`), and PR review are the load-bearing security boundaries.
2. **No fixture for the workflow's mode-0644 check.** The workflow step runs `stat -c '%a'` on the allowlist file. The check is not unit-tested here (would require GitHub Actions emulation). Operator verification: `git ls-files -s tools/check-no-direct-llm-fetch.allowlist` shows mode `100644`.
3. **Embeddings are exempt.** `flatline-semantic-similarity.sh::get_embedding` is in the allowlist. AC-1.4 reads "every direct-API path"; the embeddings call is technically a direct-API path. We accept this as documented-out-of-scope (cheval has no `embed()` method on the adapter base). When cheval adds embeddings, remove the entry from the allowlist; the gate will then enforce migration.
4. **Block-comment skip uses a single-line heuristic.** TS `/* ... */` and Python `""" ... """` strings spanning multiple lines are not tracked as a state — the scanner only skips lines that START with `*`. A URL on a line ending a block comment (e.g., `*/ var u = "https://api.openai.com/...";`) would not be detected as commented. The fix is a full state machine; given the rarity of this anti-pattern in production code, deferred.
5. **`call_api_with_retry` retry policy on `lib-curl-fallback.sh` is allowlisted.** This means the legacy `call_api()` direct curl in lib-curl-fallback.sh is permitted indefinitely. A future cycle should migrate gpt-review-api through `model-invoke` end-to-end (similar to T1.6 but for gpt-review-api callers).
