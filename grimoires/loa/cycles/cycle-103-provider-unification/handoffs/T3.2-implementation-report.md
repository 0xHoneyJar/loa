## Tasks Completed

### T3.2 — Observed-streaming audit field

**Three layers wired:**

1. **Adapters set `metadata['streaming']` at completion.** Each adapter has separate streaming + non-streaming branches. Both branches construct CompletionResult; the streaming branch sets `metadata['streaming'] = True` while the non-streaming branch sets `False`. For adapters that already populate `metadata` from a downstream parser (anthropic, openai streaming paths), the pattern is dict-merge (`_meta = dict(result.metadata or {}); _meta["streaming"] = True`) to preserve other metadata fields. For adapters that construct metadata from scratch in the non-streaming path, the pattern is `metadata={"streaming": False}` or `metadata={**existing, "streaming": False}`.

2. **cheval.py captures + propagates.** `cmd_invoke` was already passing most state through `_modelinv_state`. T3.2 adds:
   - `_modelinv_state["streaming"] = None` to the initial dict (None = adapter didn't report → emit falls back).
   - At success-completion, `_modelinv_state["streaming"] = _result_meta.get("streaming")`.
   - `_emit_modelinv(streaming=_modelinv_state["streaming"])` propagates to the audit envelope.

3. **`modelinv.py` docstring documents precedence.** The existing function already had the `streaming` parameter with env fallback — T3.2 adds the inline comment explaining the AC-3.2 motivation and the silent-degradation pattern it closes.

**Test layers:**

| Test class | Tests | What it pins |
|------------|-------|--------------|
| TestStreamingArgPrecedence | 4 | adapter-supplied True wins over env False; adapter-supplied False wins over env True; None falls back to env-derived in both directions |
| TestAdapterSetsStreamingMetadata | 3 | source-text grep ensures every adapter has the streaming flag pattern in both transport branches |

Source-text greps complement the behavioral tests. If a future refactor removes the metadata-setting code from an adapter, the grep test fails — catching the regression before the env fallback masks it.

## Technical Highlights

- **DISS-001 silent-degradation pattern fully closed.** Pre-T3.2: when an operator flipped `LOA_CHEVAL_DISABLE_STREAMING=1` mid-session, in-flight streaming requests recorded `streaming=false` (env state at emit time), while their wire behavior was actually streaming. Post-T3.2: the audit field records the wire behavior. The vision-019 M1 audit query for silent-degradation now has accurate per-call data instead of a coarse env-state proxy.
- **Adapter override is one-way.** If the adapter sets `streaming=True/False`, emit uses it. If the adapter is silent (`None`), emit falls back to env. There is no way for the env to override the adapter's claim — this is intentional. The adapter is the source of truth for which transport ran; the env is the source of truth for which transport SHOULD run.
- **`_streaming_active()` helper preserved unchanged.** Other callers of `_streaming_active()` (e.g., legacy emit paths, async/poll paths) continue to use it. T3.2 doesn't change the fallback semantics — only adds a higher-precedence override.
- **Source-text tests defend against silent removal.** A behavioral test for adapter metadata propagation would need to invoke each adapter end-to-end. A source-text grep is cheaper and catches the most common regression pattern (someone reverts the metadata line during a future refactor). Mirrors the T1.8 / T1.11 pattern.

## Known Limitations

1. **No behavioral test that adapters DO populate the field at runtime.** The TestAdapterSetsStreamingMetadata tests use grep-based source-text checks, not behavioral tests. A behavioral test would require mocking the HTTP transport and verifying the CompletionResult's `metadata['streaming']` value. The source-text check catches the most common regression class (line removal) and is cheap. A future cycle could add full-behavioral tests if drift becomes observable.
2. **Bedrock adapter not touched.** T3.2 covers anthropic / openai / google. Bedrock has its own path (`bedrock_adapter.py`) and may or may not have a streaming/non-streaming branch — out of cycle-103 scope. If bedrock is used in production, its CompletionResult would lack `metadata['streaming']` and emit would fall back to env-derived. Documented for future cycle.
3. **Headless adapters not touched.** `claude_headless_adapter.py`, `codex_headless_adapter.py`, `gemini_headless_adapter.py` — same disposition as bedrock. Out of cycle-103 scope.
4. **`_streaming_active()` global behavior unchanged.** Callers that read `_streaming_active()` directly (not through the emit function) continue to see the env-derived value. T3.2 doesn't change `_streaming_active()` semantics, only adds a higher-precedence override at the emit boundary.
