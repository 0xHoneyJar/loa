## Tasks Completed

### T1.2 — Implement ChevalDelegateAdapter (spawn-mode)

**File:** `.claude/skills/bridgebuilder-review/resources/adapters/cheval-delegate.ts` (~290 lines)

**Approach:**

1. **Class shape per SDD §5.3:** `ChevalDelegateAdapter implements ILLMProvider`, constructor takes `{ model, timeoutMs?, mockFixtureDir?, mode?, agent?, ... }`. Test-only hooks (`chevalScript`, `pythonBin`, `spawnFn`) are explicit constructor options — no global mocking, no `jest.spyOn`-style intrusion.
2. **CLI invocation:** writes systemPrompt + userPrompt to a per-call `mkdtempSync` directory, passes paths via `--system <path>` and `--input <path>`. Argv also carries `--agent`, `--model`, `--max-tokens`, `--output-format json`, `--json-errors`, `--timeout <seconds>`. The temp-dir is cleaned up in `finally` (best-effort; cleanup failure does not mask the real error).
3. **Exit-code translation:** `translateExitCode()` is exported so the table is grep-able from a code review. Exit 1 split by stderr last-line JSON `code` (cheval emits `_error_json` with the class name); RATE_LIMITED → `RATE_LIMITED`, everything else → `PROVIDER_ERROR`.
4. **Timeout escalation:** `setTimeout(timeoutMs, SIGTERM)` + nested `setTimeout(5000ms, SIGKILL)` if `child.exitCode === null` after SIGTERM.
5. **Partial-stdout handling:** `parseStdout()` returns null on JSON parse failure → throws `PROVIDER_ERROR` with `MalformedDelegateError` in the message (retry-eligible per AC-1.9 (c)). Same path for exit-0-but-missing-`content`.
6. **Credential-handoff:** `spawn(..., { env: process.env })` — child inherits parent env. No argv mutation. No stdin write. Cheval-side redactor runs before stderr surfaces to TS.
7. **LOA_BB_FORCE_LEGACY_FETCH=1 trip:** constructor checks `process.env.LOA_BB_FORCE_LEGACY_FETCH === "1"` and throws `LLMProviderError(INVALID_REQUEST, ...)` with a pointer to the (Sprint-1-deliverable) `cheval-delegate-architecture.md` runbook (T1.10). Tripping in the constructor (not at `generateReview()` call time) means the wrong-env case fails fast at adapter-factory time, before any spawn.

**Test coverage:** `__tests__/cheval-delegate.test.ts` — 35 tests in 6 describe-suites:

| Suite | Tests | Purpose |
|-------|-------|---------|
| constructor | 3 | model required; daemon-mode rejected; LOA_BB_FORCE_LEGACY_FETCH=1 trips with guided-rollback message |
| request marshaling | 6 | argv shape; --mock-fixture-dir passthrough (set + unset); AC-1.8 keys never in argv; empty prompts; bad maxOutputTokens |
| success path | 2 | parse cheval JSON output; fall back to constructor model when cheval omits |

| translateExitCode table-pinned | 11 | direct-call coverage so the table is review-traceable |

**Test injection design:** the adapter exposes `spawnFn` as a constructor option for tests. Tests pass a fake that scripts stdout/stderr/exit-code per scenario. This keeps the tests hermetic (no real Python invocation, no real api keys) and makes the SDD §5.3 table mechanically pinned. The same `spawnFn` field is the seam T1.5 will use to test the `--mock-fixture-dir` end-to-end behavior.

## Technical Highlights

- **Karpathy-aligned scope:** T1.2 ships only `cheval-delegate.ts` + tests. Did NOT touch `adapter-factory.ts` (T1.4), did NOT modify `entry.sh` (T1.8), did NOT extend cheval.py (T1.5). Surgical change.
- **Per-call temp-dir, not per-instance:** `mkdtempSync` creates a fresh directory each `generateReview` call, removed in `finally`. Concurrent calls don't share filenames. Cleanup is best-effort (rmSync `force: true`) so a stale FD does not mask the real provider error.
- **No `unref()` on timers:** the killTimer must keep the event loop alive — Node test runner cancels pending promises if the loop drains. (Discovered during test run; first iteration used `.unref()` which caused 12 cancelled tests. Removed; all 35 pass.) Production callers always have a live child-process FD keeping the loop alive, so the timer is bounded by the timeout anyway.
- **Exit-1 split via stderr JSON:** cheval emits `_error_json` on stderr with `code` matching the cheval-side exception class. The delegate reads the **last** JSON-decodable line of stderr (not necessarily the very last line — stderr may contain non-JSON diagnostics interleaved). `lastJsonLine()` walks backwards looking for a JSON object.
- **Stderr preview is capped:** non-JSON-error stderr is truncated to 256 chars and the last line only. Cheval's own redactor (`lib/log-redactor.sh` from cycle-099) runs before the bytes reach TS, but the cap is defense-in-depth against accidental large-payload echo in user-facing error messages.

## Known Limitations

1. **Daemon-mode rejected at construction.** Sprint 1 T1.3 is descoped per the T1.1 benchmark (worst p95=126ms vs 1000ms threshold). The constructor throws if `mode: "daemon"` is passed; the SDD §5.3 type-level `mode?: "spawn" | "daemon"` field stays so cycle-104+ can re-enable without a breaking change to the constructor shape.
2. **`agent` constructor option extends SDD §5.3.** Cheval requires `--agent` for binding resolution. SDD §5.3's constructor signature does not include `agent`. T1.2 adds it as an optional field with default `"reviewing-code"`. T1.4 (adapter-factory migration) will pass per-provider agent names. The SDD ought to be updated to reflect this; flagged for Sprint 1 review.
3. **--mock-fixture-dir is passthrough only.** T1.5 lands the cheval-side `--mock-fixture-dir` flag handling. Until T1.5 lands, passing `mockFixtureDir` causes cheval to error with `INVALID_INPUT` — which is the correct signal that fixture-mode is requested but unavailable.
4. **Cheval CLI shape diverges from SDD §5.1 in two ways:**
   - SDD §5.1 says `--timeout-seconds`; actual CLI is `--timeout` (integer seconds). Delegate uses the actual flag.
   - SDD §5.1 says `--json-out`; actual CLI is `--output-format json`. Delegate uses the actual flag.
   - Neither divergence has a behavioral consequence — the CLI works as designed; the SDD's prose is mildly aspirational. Flagged for Sprint 1 SDD-correction sweep.
5. **No retry loop in the delegate.** Cheval already runs its retry policy (`loa_cheval/routing/retry.py`) before exiting with `RETRIES_EXHAUSTED` → exit 1. Adding a TS-side retry would double-count attempts. The delegate surfaces typed errors and lets BB's caller (or the operator) decide whether to retry.
